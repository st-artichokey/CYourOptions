## C# Patterns Walkthrough

### Add XML doc comments and review modern C# patterns

Added `///` XML documentation to all public APIs. Walked through key C# patterns used in the codebase:

- `required` + `init` — immutable objects without constructor boilerplate
- `[]` collection expressions — concise empty collection initialization
- `=>` expression-bodied members — single-expression computed properties
- `readonly` fields + `IReadOnlyList<T>` — encapsulation via restrictive exposure
- Nullable reference types (`?`) — compile-time null safety
- `[^1]` index-from-end — ergonomic last-element access
- `static readonly` config — reusable objects (JsonSerializerOptions best practice)
- `??` null-coalescing throw — concise null guards
- Lambda expressions — functional-style LINQ operations
- Specific exception types — `ArgumentException`, `ArgumentOutOfRangeException`, `InvalidOperationException`

**Sources:**
- [C# nullable reference types](https://learn.microsoft.com/en-us/dotnet/csharp/nullable-references)
- [System.Text.Json best practices](https://learn.microsoft.com/en-us/dotnet/standard/serialization/system-text-json/configure-options)
